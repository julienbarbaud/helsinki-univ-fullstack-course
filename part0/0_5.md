Sequence diagram for loading the single-page style app at https://studies.cs.helsinki.fi/exampleapp/spa

(NB: This is pretty much exactly the same as the previous https://studies.cs.helsinki.fi/exampleapp/notes app since almost only the submitting of a new note differs)

```mermaid
sequenceDiagram
  participant browser
  participant server

  browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/spa
  activate server
  server-->>browser: HTML document
  deactivate server

  Note right of browser: browser finds reference to main.css in the HTML

  browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/main.css
  activate server
  server-->>browser: main.css
  deactivate server

  Note right of browser: browser finds reference to spa.js in the HTML

  browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/spa.js
  activate server
  server-->>browser: spa.js
  deactivate server

  Note right of browser: spa.js is executed and fetches data.json

  browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/data.json
  activate server
  server-->>browser: data.json
  deactivate server

  Note right of browser: callback function in main.js stores the data.json in the local variable notes and calls the redrawNotes() function ot display the page
```
